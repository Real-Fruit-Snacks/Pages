NAME

gh-workflow-run
- Run a workflow by creating a workflow_dispatch event

SYNOPSIS

gh workflow
run [ | ]
[flags]

DESCRIPTION

Create a
workflow_dispatch event for a given workflow.

This command
will trigger GitHub Actions to run a given workflow file.
The given workflow file must support an
on.workflow_dispatch trigger in order to be run in
this way.

If the workflow
file supports inputs, they can be specified in a few
ways:

&bull;

Interactively

&bull;

Via -f/--raw-field or -F/--field flags

&bull;

As JSON, via standard input

OPTIONS

-F,
--field 

Add a string parameter in
key=value format, respecting @ syntax (see "gh help
api").

--json

Read workflow inputs as JSON via STDIN

-f, --raw-field

Add a string parameter in
key=value format

-r, --ref

The branch or tag name which
contains the version of the workflow file you&rsquo;d like
to run

OPTIONS INHERITED FROM PARENT COMMANDS

-R,
--repo 

Select another repository using
the [HOST/]OWNER/REPO format

EXAMPLE

# Have gh
prompt you for what workflow you&rsquo;d like to run and
interactively collect inputs 
$ gh workflow run

# Run the
workflow file &rsquo;triage.yml&rsquo; at the remote&rsquo;s
default branch 
$ gh workflow run triage.yml

# Run the
workflow file &rsquo;triage.yml&rsquo; at a specified ref

$ gh workflow run triage.yml --ref my-branch

# Run the
workflow file &rsquo;triage.yml&rsquo; with command line
inputs 
$ gh workflow run triage.yml -f name=scully -f
greeting=hello

# Run the
workflow file &rsquo;triage.yml&rsquo; with JSON via
standard input 
$ echo &rsquo;{"name":"scully",
"greeting":"hello"}&rsquo; | gh workflow
run triage.yml --json
