NAME

autoconf -
Generate configuration scripts

SYNOPSIS

autoconf
[OPTION]... [TEMPLATE-FILE]

DESCRIPTION

Generate a
configuration script from a TEMPLATE-FILE if given, or
&rsquo;configure.ac&rsquo; if present, or else
&rsquo;configure.in&rsquo;. Output is sent to the standard
output if TEMPLATE-FILE is given, else into
&rsquo;configure&rsquo;.

Operation
modes: 
-h, --help

print this help, then exit

-V, --version

print version number, then
exit

-v, --verbose

verbosely report processing

-d, --debug

don&rsquo;t remove temporary
files

-f, --force

consider all files obsolete

-o,
--output=FILE

save output in FILE (stdout is
the default)

-W,
--warnings=CATEGORY

report the warnings falling in
CATEGORY [syntax]

Warning
categories include: 
&rsquo;cross&rsquo;

cross compilation issues

&rsquo;obsolete&rsquo;

obsolete constructs

&rsquo;syntax&rsquo;

dubious syntactic
constructs

&rsquo;all&rsquo;

all the warnings

&rsquo;no-CATEGORY&rsquo;

turn off the warnings on
CATEGORY

&rsquo;none&rsquo;

turn off all the warnings

&rsquo;error&rsquo;

warnings are error

The environment
variables &rsquo;M4&rsquo; and &rsquo;WARNINGS&rsquo; are
honored.

Library
directories: 
-B, --prepend-include=DIR

prepend directory DIR to search
path

-I,
--include=DIR

append directory DIR to search
path

Tracing:

-t, --trace=MACRO[:FORMAT]

report the list of calls to
MACRO

-i,
--initialization

also trace Autoconf&rsquo;s
initialization process

In tracing
mode, no configuration script is created. FORMAT defaults to
&rsquo;$f:$l:$n:$%&rsquo;; see &rsquo;autom4te
--help&rsquo; for information about FORMAT.
