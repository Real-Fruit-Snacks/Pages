NAME

Lzip &minus;
data compressor based on the LZMA algorithm

SYNOPSIS

lzip
[options] [files]

DESCRIPTION

Lzip &minus; A
data compressor based on the LZMA algorithm.

OPTIONS

&minus;h,
&minus;&minus;help

display this help and exit

&minus;V,
&minus;&minus;version

output version information and
exit

&minus;b,
&minus;&minus;member&minus;size=

set member size limit in
bytes

&minus;c,
&minus;&minus;stdout

send output to standard
output

&minus;d,
&minus;&minus;decompress

decompress

&minus;f,
&minus;&minus;force

overwrite existing output
files

&minus;k,
&minus;&minus;keep

keep (don&rsquo;t delete) input
files

&minus;m,
&minus;&minus;match&minus;length=

set match length limit in bytes
[80]

&minus;o,
&minus;&minus;output=

if reading stdin, place the
output into 

&minus;q,
&minus;&minus;quiet

suppress all messages

&minus;s,
&minus;&minus;dictionary&minus;size=

set dictionary size limit in
bytes [8MiB]

&minus;S,
&minus;&minus;volume&minus;size=

set volume size limit in
bytes

&minus;t,
&minus;&minus;test

test compressed file
integrity

&minus;v,
&minus;&minus;verbose

be verbose (a 2nd
&minus;v gives more)

&minus;1 ..
&minus;9

set compression level [default
6]

&minus;&minus;fast

alias for &minus;1

&minus;&minus;best

alias for &minus;9

If no file
names are given, lzip compresses or decompresses from
standard input to standard output. Numbers may be followed
by a multiplier: k = kB = 10^3 = 1000, Ki = KiB = 2^10 =
1024, M = 10^6, Mi = 2^20, G = 10^9, Gi = 2^30, etc...
