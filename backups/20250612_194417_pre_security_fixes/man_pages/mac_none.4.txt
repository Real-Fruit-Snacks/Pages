NAME

mac_none — null MAC
policy module

SYNOPSIS

To compile the null policy into
your kernel, place the following lines in your kernel
configuration file:

options MAC

options MAC_NONE

Alternately, to
load the none module at boot time, place the following line
in your kernel configuration file:

options
MAC

and in
loader.conf(5):

mac_none_load="YES"

DESCRIPTION

The mac_none policy
module implements a none MAC policy that has no effect on
access control in the system. Unlike mac_stub(4), none of
the MAC entry points are defined.

Label
Format 
No labels are defined for mac_none.

HISTORY

The mac_none policy
module first appeared in FreeBSD 5.0 and was developed
by the TrustedBSD Project.

BUGS

While the MAC Framework design
is intended to support the containment of the root user, not
all attack channels are currently protected by entry point
checks. As such, MAC Framework policies should not be relied
on, in isolation, to protect against a malicious privileged
user.
