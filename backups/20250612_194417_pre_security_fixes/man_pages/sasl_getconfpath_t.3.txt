NAME

sasl_getconfpath_t
- Cyrus SASL documentation

SYNOPSIS

#include

int
sasl_getconfpath_t(void *context, char ** path);

DESCRIPTION

int
sasl_getconfpath_t(void *context, char **path);

sasl_getconfpath_t() is
used if the application wishes to use a different location
for the SASL configuration files. If this callback is not
used SASL will either use the location in the environment
variable SASL_CONF_PATH (provided we are not SUID or SGID)
or /etc/sasl2 by default.

RETURN VALUE

SASL callback
functions should return SASL return codes. See sasl.h for a
complete list. SASL_OK indicates success.

Other return
codes indicate errors and should be handled.
