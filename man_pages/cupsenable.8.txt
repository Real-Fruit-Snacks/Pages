NOME
       cupsdisable, cupsenable - pára/arranca impressoras e classes

RESUMO
       cupsdisable  [  -E  ]  [ -U username ] [ -c ] [ -h server[:port] ] [ -r
       reason ] [ --hold ] destination(s)
       cupsenable [ -E ] [ -U username ] [ -c ] [ -h server[:port] ]  [  --re‐
       lease ] destination(s)

DESCRIÇÃO
       cupsenable  inicia as impressoras ou classes nomeadas enquanto cupsdis‐
       able pára as impressoras ou classes nomeadas.

OPÇÕES
       As seguintes opções podem ser usadas:

       -E   Força encriptação da ligação ao servidor.

       -U username
            Usa o nome de utilizador especificado ao ligar ao servidor.

       -c   Cancela todos os trabalhos no destino nomeado.

       -h server[:port]
            Usa o servidor e porto especificado.

       --hold
            Retém os trabalhos restantes na impressora nomeada. Útil para per‐
            mitir que o trabalho actual termine antes de executar manutenção.

       -r "reason"
            Define a mensagem associada ao estado de paragem. Se nenhuma razão
            for especificada, então a mensagem é definida para "Razão  Descon‐
            hecida".

       --release
            Liberta  trabalhos pendentes para impressão. Use após correr cups‐
            disable com a opção --hold para resumir a impressão.

DE ACORDO COM
       Ao contrário do sistema de impressão System V, o CUPS  permite  que  os
       nomes  das  impressoras contenham qualquer caractere imprimível excepto
       SPACE, TAB, "/", ou "#". Também, os nomes das impressoras e classes não
       são sensíveis a maiúsculas/minúsculas.

       As versões do System V destes comandos são disable e enable, respetiva‐
       mente. Eles foram renomeados para evitar conflitos com os comandos  em‐
       butidos no bash(1) com os mesmos nomes.

       As  versões  CUPS  do  disable  e  enable podem pedir ao utilizador uma
       palavra passe de acesso dependendo da configuração do  sistema  de  im‐
       pressão.  Isto  difere  das versões do System V que requerem que seja o
       utilizador root a executar estes comandos.
