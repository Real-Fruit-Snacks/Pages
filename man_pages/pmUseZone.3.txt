NAME

pmUseZone
- re-establish a reporting timezone

C SYNOPSIS

#include

int
pmUseZone(const int tz_handle);

cc ...
-lpcp

DESCRIPTION

The current
reporting timezone effects the timezone used by
pmCtime(3) and pmLocaltime(3).

The argument
tz_handle identifies a reporting timezone as
previously established by a call to pmNewZone(3) or
pmNewContextZone(3), and this becomes the current
reporting timezone.

DIAGNOSTICS

A return value
less than zero indicates the value of tz_handle is
not legal.

SEE ALSO

PMAPI(3),
pmCtime(3), pmLocaltime(3),
pmNewContextZone(3), pmNewZone(3) and
pmWhichZone(3).
