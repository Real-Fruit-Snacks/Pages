NAME

mail &minus;
the standard /usr/bin/mail interface

SYNOPSIS

mail
[OPTION...] [address...]

DESCRIPTION

GNU mail
&minus;&minus; the standard /usr/bin/mail
interface

&minus;a,
&minus;&minus;append=HEADER: VALUE Append
given header to the message being sent

&minus;e, &minus;&minus;exist

Return true if mail exists

&minus;E,
&minus;&minus;exec=COMMAND

Execute COMMAND

&minus;f,
&minus;&minus;file[=URL]

Operate on given mailbox URL
(default ~/mbox)

&minus;F,
&minus;&minus;byname

Save messages according to
sender

&minus;H,
&minus;&minus;headers

Write a header summary and
exit

&minus;i,
&minus;&minus;ignore

Ignore interrupts

&minus;n,
&minus;&minus;norc

Do not read the system mailrc
file

&minus;N,
&minus;&minus;nosum

Do not display initial header
summary

&minus;p,
&minus;&minus;print

Print all mail to standard
output

&minus;q,
&minus;&minus;quit

Cause interrupts to terminate
program

&minus;r,
&minus;&minus;read

Same as &minus;p

&minus;s,
&minus;&minus;subject=SUBJ

Send a message with a Subject
of SUBJ

&minus;t,
&minus;&minus;to

Precede message by a list of
addresses

&minus;u,
&minus;&minus;user=USER

Operate on USER&rsquo;s
mailbox

Common
options

&minus;&minus;license

Print license and exit

&minus;&minus;external&minus;locker=PATH
Set full path to the external locker program

&minus;&minus;lock&minus;expire&minus;timeout=SECONDS 

Number of seconds after which
the lock expires

&minus;&minus;lock&minus;flags=FLAGS

Default locker flags
(E=external, R=retry, T=time, P=pid)

&minus;&minus;lock&minus;retry&minus;count=NUMBER

Set the maximum number of times
to retry acquiring the lockfile

&minus;&minus;lock&minus;retry&minus;timeout=SECONDS

Set timeout for acquiring the
lockfile

&minus;m,
&minus;&minus;mail&minus;spool=URL

Use specified URL as a
mailspool directory

&minus;&minus;mailbox&minus;type=PROTO

Default mailbox type to use

&minus;&minus;tls[=BOOL]

Enable TLS support

&minus;?,
&minus;&minus;help

give this help list

&minus;&minus;usage

give a short usage message

&minus;V,
&minus;&minus;version

print program version

Mandatory or
optional arguments to long options are also mandatory or
optional for any corresponding short options.
