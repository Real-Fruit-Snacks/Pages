NAME

sq key expire -
Change expiration times

SYNOPSIS

sq key
expire [OPTIONS] EXPIRY FILE

DESCRIPTION

Change
expiration times.

Keys and their
individual subkeys can expire. This subcommand changes or
clears the expiration times.

By default, the
expiration time of the entire key is changed. To change the
expiration of only some of the subkeys, use the
&rsquo;--subkey&rsquo; option.

OPTIONS

Subcommand
options 
-B, --binary

Emit binary data

-o,
--output=FILE

Write to FILE or stdout if
omitted

--subkey=SUBKEY

Change expiration of this
subkey, not the entire key

EXPIRY

Define EXPIRY for the key as
ISO 8601 formatted string or custom duration. If an ISO 8601
formatted string is provided, the validity period reaches
from the reference time (may be set using
&rsquo;--time&rsquo;) to the provided time. Custom durations
starting from the reference time may be set using
&rsquo;N[ymwds]&rsquo;, for N years, months, weeks, days, or
seconds. The special keyword &rsquo;never&rsquo; sets an
unlimited expiry.

FILE

Read from FILE or stdin if omitted

Global
options 
See sq(1) for a description of the global
options.

EXAMPLES

Make
Alice&rsquo;s key expire in a year.

sq key expire
1y alice-secret.pgp

Make
Alice&rsquo;s key never expire.

sq key expire
never alice-secret.pgp

Make
Bob&rsquo;s authentication subkey expire in six months.

sq key expire
6m --subkey 6AEACDD24F896624 bob-secret.pgp
