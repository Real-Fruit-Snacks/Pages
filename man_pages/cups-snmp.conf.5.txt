NOME
       snmp.conf - ficheiro de configuração de snmp para cups (descontinuado)

DESCRIÇÃO
       O  ficheiro  snmp.conf  configura  como os backends de rede standard do
       CUPS (http, https, ipp, ipps, lpd, snmp, e socket) acedem a  informação
       da impressora usando SNMPv1 e está normalmente localizado no directório
       /etc/cups. Cada linha no  ficheiro  pode  ser  uma  directiva  de  con‐
       figuração,  uma  linha  vazia,  ou  um comentário. As linhas comentário
       começam com o caractere #.

       As directivas Community e DebugLevel são usadas por todos os  backends.
       A "remainder" só se aplica apenas ao backend SNMP - cups-snmp(8).

DIRECTIVAS
       As  seguintes  directivas  são  compreendidas pelos backends de rede do
       CUPS:

       Address @IF(name)

       Address @LOCAL

       Address address
            Envia consultas de difusão SNMP  (para  descoberta)  para  os  en‐
            dereço(s)  especificados.  Não existe predefinição para o endereço
            de difusão.

       Community name
            Especifica o nome de comunidade a usar. Apenas um  único  nome  de
            comunidade pode ser especificado. O nome de comunidade predefinido
            é "public". Se nenhum nome for especificado, todas as funções SNMP
            são desactivadas.

       DebugLevel number
            Especifica o nível de relatórios de 0 (nada) até 3 (tudo). Tipica‐
            mente apenas usado para depuração (daí o nome). O nível  de  debug
            predefinido é 0.

       DeviceURI "regular expression" device-uri [... device-uri]
            Especifica  um  ou  mais  URIs de dispositivo que devem ser usados
            para uma determinada string de marca e modelo. A expressão regular
            é  usada para corresponder à marca e modelo detetado, e as strings
            de   URI   de   dispositivo    têm    de    estar    no    formato
            "scheme://%s[:port]/[path]",  onde  "%s"  representa o endereço ou
            nome de máquina detetado. Não  existem  regras  predefinidas  para
            correspondência de URI de dispositivo.

       HostNameLookups on

       HostNameLookups off
            Especifica  se  os  endereços de impressoras devem ser convertidos
            para nomes de máquinas ou deixados como endereços IP numéricos.  A
            predefinição é "off".

       MaxRunTime seconds
            Especifica  o número máximo de segundos em que um backend SNMP irá
            sondar a rede por impressoras. A predefinição é  120  segundos  (2
            minutos).

NOTAS
       Os backends do CUPS estão descontinuados e não irão ser mais suportados
       num futuro lançamento do CUPS. Impressoras que não suportem  IPP  podem
       ser suportadas usando aplicações como a ippeveprinter(1).
