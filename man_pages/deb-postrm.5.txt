NAMN
       deb-postrm - paketutvecklarskript som körs efter borttagning

SYNOPS
       DEBIAN/postrm

BESKRIVNING
       Ett paket kan utföra flera åtgärder efter borttagning via
       utvecklarskript, genom att ta med en körbar fil postrm i sitt styrarkiv
       (dvs. DEBIAN/postrm när paketet skapas).

       Skriptet kan anropas på följande sätt:

       postrm remove
           Efter att paketet tagits bort.

       postrm purge
           Efter att paketet har raderats helt.

       old-postrm upgrade ny-version
           Efter att paketet har uppgraderats.

       ny-postrm failed-upgrade gammal-version ny-version
           Om ovanstående upgrade-anrop misslyckas.

           ny-version sänds endast sedan dpkg 1.18.5.

       postrm disappear överskrivande-paket överskrivande-version
           Efter att alla filer i paketet har ersatts.

       ny-postrm abort-install
           Om preinst misslyckas under install (installation).

       ny-postrm abort-install gammal-version ny-version
           Om preinst misslyckas install (installation) vid uppgradering av
           borttagna paket.

           ny-version sänds endast sedan dpkg 1.18.5.

       ny-postrm abort-upgrade gammal-version ny-version
           Om preinst misslycas vid upgrade (uppgradering).

           ny-version sänds endast sedan dpkg 1.18.5.
