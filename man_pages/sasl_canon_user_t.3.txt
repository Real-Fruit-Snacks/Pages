sasl_canon_user_t is the
callback for an application-supplied user canonicalization
function. This function is subject to the requirements that
all user canonicalization functions are: It must copy the
result into the output buffers, but the output buffers and
the input buffers may be the same. 
Parameters

&bull;

context &ndash; context from the callback
record 

&bull;

user &ndash; un-canonicalized username

&bull;

ulen &ndash; length of user

&bull;

flags &ndash; Either SASL_CU_AUTHID (indicating
the authentication ID is being canonicalized) or
SASL_CU_AUTHZID (indicating the authorization ID is to be
canonicalized) or a bitwise OR of the the two.

&bull;

user_realm &ndash; Realm of authentication.

&bull;

out_user &ndash; The output buffer for the
canonicalized username

&bull;

out_umax &ndash; Maximum length for out_user

&bull;

out_ulen &ndash; Actual length of out_user

Returns

SASL_OK indicates
success. See sasl_errors(3) for a full list of SASL
error codes.
